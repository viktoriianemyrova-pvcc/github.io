Data relationdhips:
  1. One-to-one - a link between the information in two tables, where each record in each table only appears once. For example, there might be a one-to-one relationship between employees and the cars they drive.
  2. One-to-many - one record in a table can be associated with one or more records in another table. For example, each customer can have many sales orders.
  3. Many-to-many - when one or more items in one table can have a relationship to one or more items in another table. For example: Your Order table contains orders placed by multiple customers (who are listed in the Customers table), and a customer may place more than one order.
