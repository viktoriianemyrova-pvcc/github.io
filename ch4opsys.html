Chapter 4 Terms

Network Classifications
  • LAN - a collection of devices connected together in one physical location, such as a building, office, or home.
  • MAN - a computer network that connects computers within a metropolitan area.
  • WAN - a large computer network that connects groups of computers over large distances.
Network models
  • Bus network - a local area network (LAN) topology in which each node -- a workstation or other device -- is connected to a main cable or link called a bus.
  • Star network - very host is connected to a central hub. In its simplest form, one central hub acts as a conduit to transmit messages. The star network is one of the most common computer network topologies.
Protocols
  • CSMA/CD Ethernet - a medium access control (MAC) method used most notably in early Ethernet technology for local area networking.
  • CSMA/CA Wifi - a protocol for carrier transmission in 802.11 networks. It was developed to minimize the potential of a collision occurring when two or more stations send their signals over a data link layer.
Combining network:
  • Repeater - a device that amplifies or adds to incoming electrical signals and retransmits them, in order to compensate for transmission losses.
  • Bridge - a computer networking device that creates a single, aggregate network from multiple communication networks or network segments.
  • Switch - connects devices in a network to each other, enabling them to talk by exchanging data packets. Switches can be hardware devices that manage physical networks or software-based virtual devices.
  • Router - a device that connects two or more packet-switched networks or subnetworks.
  • Gateway - the Internet Connection Sharing feature allows a computer to act as a gateway by offering a connection between the Internet and an internal network.
Client/server model
  • P2P - a decentralized platform whereby two individuals interact directly with each other, without intermediation by a third party.
  • Distributed system - a collection of independent components located on different machines that share messages with each other in order to achieve common goals.
  • Cloud computing - streaming platforms for audio or video, where the actual media files are stored remotely.
Internet Addressing
  • Tier 1 - an Internet Protocol (IP) network that can reach every other network on the Internet solely via settlement-free interconnection (also known as settlement-free peering).
  • Tier 2 - a service provider that utilizes a combination of paid transit via Tier 1 ISPs and peering with other Tier 2 ISPs to deliver Internet traffic to end customers through Tier 3 ISPs. Tier 2 ISPs are typically regional or national providers.
  • Tier 3 - primarily engaged in delivering Internet access to end customers. Tier 3 ISPs focus on local business and consumer market conditions.
IP address
  • ICANN
  • DNS - turns domain names into IP addresses, which browsers use to load internet pages. Every device connected to the internet has its own IP address, which is used by other devices to locate the device.
  • Top-Level domain (TLD)- represents the first stop after the root zone. In simpler terms, a TLD is everything that follows the final dot of a domain name.
  • Country-code TLDs
Internet Protocols
  • FTP - a standard network protocol used for the transfer of files from one host to another over a TCP-based network, such as the Internet.
  • HTTP - is the foundation of the World Wide Web, and is used to load webpages using hypertext links.
  • SMTP - an application that is used to send, receive, and relay outgoing emails between senders and receivers.
World Wide Web
  • URL - a unique identifier used to locate a resource on the Internet.
  • Client-side activities - everything in a web application that is displayed or takes place on the client (end user device). This includes what the user sees, such as text, images, and the rest of the UI, along with any actions that an application performs within the user's browser.
  • Server-side activities - everything that happens on the server, instead of on the client. In the past, nearly all business logic ran on the server side, and this included rendering dynamic webpages, interacting with databases, identity authentication, and push notifications.
Internet four-layer software
  • Application layer - an abstraction layer that specifies the shared communication protocols and interface methods used by hosts in a communications network.
  • Transport layer - accept data from the session layer, split it up into smaller units if need be, pass these to the Network layer, and ensure that all the pieces arrive correctly at the other end.
  • Network layer - the part of the Internet communications process where these connections occur, by sending packets of data back and forth between different networks.
  • Link layer - The data link layer, or layer 2, is the second layer of the seven-layer OSI model of computer networking. This layer is the protocol layer that transfers data between nodes on a network segment across the physical layer.
Internet is:
  • Packet-switching network - the transfer of small pieces of data across various networks.
  • Fault tolerant - a process that enables an operating system to respond to a failure in hardware or software.
  • Redundant - he principle of using backup network resources to minimize or prevent downtime if there's a power outage, hardware malfunction, human error, system failure, or cyber attack.
Cybersecurity
  • Malware - any program or file that is intentionally harmful to a computer, network or server.
  • Virus - an infectious microbe consisting of a segment of nucleic acid (either DNA or RNA) surrounded by a protein coat.
  • Worm - a type of malware or malicious software that can replicate rapidly and spread across devices within a network.
  • Trojan horse - a type of malware that downloads onto a computer disguised as a legitimate program.
  • Spyware - a type of malicious software -- or malware -- that is installed on a computing device without the end user's knowledge.
  • Phishing - the practice of sending fraudulent communications that appear to come from a legitimate and reputable source, usually through email and text messaging.
  • DDoS attack - a cybercrime in which the attacker floods a server with internet traffic to prevent users from accessing connected online services and sites.
Protection
  • Firewall - a digital security system that checks all incoming and outgoing traffic on a network according to a defined set of rules.
  • Spam filter - designed to identify emails that attackers or marketers use to send unwanted or dangerous content.
  • Proxy server - acts as a gateway between you and the internet. It's an intermediary server separating end users from the websites they browse. Proxy servers provide varying levels of functionality, security, and privacy depending on your use case, needs, or company policy.
  • Antivirus software - a security program designed to prevent, detect, search and remove viruses and other types of malware from computers, networks and other devices.
  • Cryptography - the process of hiding or coding information so that only the person a message was intended for can read it.
  • Public key encryption - a method of encrypting or signing data with two different keys and making one of the keys, the public key, available for anyone to use.
